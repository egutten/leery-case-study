Case Study: Leery

Dynamic social proof for B2B businesses

Problem

B2B businesses do not leverage dynamic forms of social proof as effectively as B2C businesses.

Solution

Overview

I came to this solution by researching and examining competitive social proof products. The most compelling to me was Proof, and it struck me how it was built only for B2C businesses. I came to the idea of adding logos to Proof's message model by noticing that the typical B2B practice is to statically post logos on a homepage. That static posting not only discourages any updates to the group of logos, it also does not bring any particular attention to them.

  • Concept

    • Show Prospective customers only the most current social proof data through a pop-up messate (the "Message"), which includes:

      • The most recent conversion event that took place (e.g. “Just signed-up for a trial”)
      • How long ago the conversion event occurred (e.g. “2 seconds ago”)
      • The logo of the customer involved in the social proof (to take advantage of a benefit particular to B2B businesses)

  • Execution

    • Offer third-party javascript that:

      • Gathers a B2B business’ (User’s) conversion data necessary to populate the Message (“Conversion Snippet”)
      • Generates the Message on load of a User’s webpage (“Message Snippet”)

User Stories

I started by writing user stories, not only to help me think through what I needed to do, but also to simulate the project planning process that might occur with a larger team.

  • As a User:

    • I want to be presented with a landing page for the Leery product that funnels me to the sign-in/log-in routes so it is easy for me to access it.
    • I want to be able to configure and preview the appearance of my Message during the sign-up flow so I can use Leery right away.

    • While logged in to the Leery User App I want to:

      • Access and be instructed how to use the pre-generated Snippets
      • Edit my original Message and create new Message configurations
      So I can control my Message content and won’t have to worry about losing Snippets.
    • I want to load the Message Snippet and Conversion Snippet so Leery can function as described.
    • I want the Message to include the most current conversion that took place, as well as the logo of the customer (“Customer”) involved so I can take advantage of the benefits of dynamic social proof.

User Flows

The diagrams below show the full use cycle of the Leery product. The colored labels (e.g. "User App") refer to the various components of the product, which are described in more detail in the System Design section below.

System Design

  • Leery Landing Page (Webflow)

    • Explains the product and Links to the User App.

  • User App (Node-React-Express)

    • An app for B2B businesses to sign up for Leery and access the user portal.
    • The user portal allows for the creation and editing of Messages, as well as continued access to Snippets.

  • Widget (Node-React)

    • Accessed through the Message Snippet, generates Messages.

  • Conversion Assist (Javascript)

    • Accessed through the Conversion Snippet, pulls Customer conversion data.

  • API (Node-Express)

    • Connects to PostgreSQL database to store:

      • User credentials
      • User conversion events
      • Customer conversion data
      • Customer's User activities (e.g. User page visits, Messages viewed, number of conversions)
